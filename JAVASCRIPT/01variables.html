1.What is a Variable ?

Variables are used to store data in JavaScript. Variables aren't the values themselves; they are containers for values. The values of the variables are allocated using the assignment operator(“=”).

  2. Initializing a variable

It simply means create a variable. for instance,
let myName; let myAge;
Here we're creating two variables called myName and myAge.

2. What are the conventions of naming a variable in JavaScript?

In JavaScript, variable names must adhere to certain conventions to ensure clarity, consistency, and maintainability of code. Here are the common conventions for naming variables in JavaScript:

Use descriptive names:

Variable names should clearly convey their purpose or the data they hold. Avoid single-letter variable names (x, y, etc.) unless they are used as loop counters or in other specific contexts where their meaning is obvious.

Example

                // Good:
                var firstName = "John";
                var itemCount = 10;

                // Bad:
                var f = "John"; // Not descriptive
                var i = 10; // Not descriptive
Use camelCase:

In JavaScript, it's conventional to use camelCase for variable names. Start with a lowercase letter, and capitalize the first letter of each subsequent word within the name. This convention applies to variable names, function names, and object property names.

Example

                var userName = "john_doe";
                var itemCount = 10;
Avoid reserved words:

Do not use JavaScript reserved words as variable names, as they have special meaning within the language.

Example

                // Bad:
                var var = "value"; // "var" is a reserved keyword
Use meaningful abbreviations:

While descriptive names are encouraged, it's acceptable to use common abbreviations as long as they are widely understood within the context of your codebase.

Example

                var maxNum = 100; // "maxNum" is an abbreviation for "maximum number"
Be consistent:

Maintain consistency in naming conventions throughout your codebase to improve readability and make it easier for others (or your future self) to understand the code

Constants:

Use uppercase letters for constants to differentiate them from regular variables. Separate multiple words with underscores.

Example

                const MAX_COUNT = 100;
Constructor functions:

When defining constructor functions for objects, use PascalCase convention.

Example

                function Car(make, model) {
                    this.make = make;
                    this.model = model;
                }
                var myCar = new Car("Toyota", "Corolla");
                 const MAX_COUNT = 100;

4. What's the difference between let and var?

In JavaScript, both let and var are used for variable declaration, but they have some key differences:

Scope:
  • var is function-scoped. This means that variables declared with var are scoped to the function in which they are declared or, if declared outside of any function, to the global scope.
  • let is block-scoped. Variables declared with let are scoped to the nearest enclosing block, whether it's a loop, an if statement, or any other block.
Hoisting:
  • Variables declared with var are hoisted to the top of their function or global scope. This means that you can access and use the variable before it's declared in the code, but its value will be undefined.
  • Variables declared with let are also hoisted, but they are not initialized until their declaration statement is evaluated. Accessing a let variable before its declaration (within the same block) results in a ReferenceError.
Global Object Property:
  • Variables declared with var become properties of the global object (window in a browser environment). This can lead to unintended consequences and potential naming conflicts.
  • Variables declared with let do not become properties of the global object, even if declared in the global scope.
Example 
                function example() {
                    console.log(x); // undefined
                    var x = 10;
                    console.log(x); // 10
                }
                
                function anotherExample() {
                    console.log(y); // ReferenceError: y is not defined
                    let y = 20;
                    console.log(y); // 20
                }
                
                example();
                anotherExample();
5. What are the ways of defining a variable in JavaScript?

In JavaScript, there are several ways to define variables, each with its own characteristics and use cases:

var:

The traditional way of defining variables in JavaScript. Variables declared with var are function-scoped or globally scoped, hoisted to the top of their scope, and can be re-declared and re-assigned.

Example

var x = 10;
let:

Introduced in ECMAScript 6 (ES6), let allows for block-scoped variables. Variables declared with let are limited to the block, statement, or expression in which they are defined. They are hoisted to the top of their block but are not initialized until their declaration is evaluated.

Example

let y = 20;
const:

Also introduced in ES6, const is used to declare constants, which are block-scoped like variables declared with let. The value of a const variable cannot be re-assigned once initialized. However, if the value is an object or array, its properties or elements can still be modified.

Example

const PI = 3.14;
Implicit declaration:

If you assign a value to a variable without declaring it using var, let, or const, JavaScript will create an implicit global variable. However, it's considered bad practice because it can lead to naming conflicts and make code harder to understand and maintain.

Example

implicitVar = "Implicit global variable";
6. What are Imports and Exports in JavaScript?

In JavaScript, imports and exports are mechanisms for modularizing code and facilitating the organization and reuse of functionality across different parts of a program or multiple programs.

Exports:
  • Exporting: This is the process of making variables, functions, or objects available for use in other JavaScript files or modules.
  • Exporting Named Entities: You can export individual variables, functions, or objects by using the export keyword followed by the name of the entity you want to export.

    Example

                            // Exporting individual entities
                        export const myVar = 10;
                        export function myFunction() { /* function code */ }
                        export class MyClass { /* class code */ }
  • Exporting Named Entities in Bulk: You can also export multiple entities at once using the export keyword with the export { entity1, entity2 } syntax.

    Example

    // Exporting multiple entities at once
                        const variable1 = 10;
                        function function1() { /* function code */ }
                        export { variable1, function1 };
  • Default Exports A module can also have a default export, which is the primary entity exported from the module. You can only have one default export per module..

    Example

                            // Default export
                        const myVar = 10;
                        export default myVar;
Imports:
  • Importing This is the process of bringing exported functionality into another JavaScript file or module.
  • Named Imports: To import specific exported entities from another module, you use the import keyword followed by braces {} containing the names of the entities you want to import.

    Example

                            import { myVar, myFunction } from './module.js';
  • Default Imports For importing the default export from a module, you use the import keyword followed by any name you choose to assign to the default export.

    Example

                            import myVar from './module.js';
  • Importing Everything: You can import all exported entities from a module using the * as syntax.

    EXample

    import * as myModule from './module.js';

Hoisting in JavaScript is a behavior where variable and function declarations are moved to the top of their containing scope during the compilation phase, before the code is executed. This means that you can use variables and functions before they are declared in your code. However, it's important to note that only the declarations are hoisted, not the initializations or assignments.

JavaScript Declarations are Hoisted

In JavaScript, a variable can be declared after it has been used.

10. Implicit Type Coercion in javascript (in detail with examples)

Implicit type coercion in JavaScript refers to the automatic conversion of values from one data type to another by the JavaScript engine during runtime. This occurs when an operation involving different data types is performed. JavaScript will attempt to convert one or both of the values to a common data type before executing the operation. Implicit type coercion can be convenient but may lead to unexpected behavior if not understood properly.

Examples of Implicit Type Coercion:

Example 1: Addition Operator (+):

                let a = 5;
                let b = "10";
                let result = a + b; // Result: "510"

In this example, JavaScript implicitly coerces the number 5 to a string to perform concatenation rather than arithmetic addition.

Example 2: Equality Operator (==):

                let num = 10;
                let str = "10";
                console.log(num == str); // Result: true

In this example, JavaScript implicitly coerces the string "10" to a number to check for equality with the numeric value 10.

Example 3: Logical Operators (&&, ||, !):

                console.log(10 && 20); // Result: 20
                console.log("" || "Hello"); // Result: "Hello"
                console.log(!true); // Result: false

In these examples, JavaScript implicitly coerces the operands to boolean values (true or false) before performing the logical operation.

Example 4: Conditional (Ternary) Operator (? :):

            let x = 10;
            let y = "5";
            let result = x > y ? "x is greater" : "y is greater"; // Result: "x is greater"

Here, JavaScript implicitly coerces the string "5" to a number to perform the comparison.

11. Is javascript a statically typed or a dynamically typed language?

JavaScript is a dynamically typed language.

In statically typed languages, variable types are explicitly declared and checked at compile time. Once a variable is declared with a specific type, its type cannot be changed

In dynamically typed languages like JavaScript, variable types are determined at runtime. You don't need to declare the type of a variable explicitly. Instead, the type of a variable is inferred based on the value assigned to it, and it can change over the course of the program's execution.

12. NaN property in JavaScript

In JavaScript, NaN is a global property representing "Not-A-Number." It indicates that a value is not a legal number. The NaN property is a special value of the global object and is of the type "number".

Characteristics of NaN:

  • Type: Despite its name, NaN is a number data type.
  • Indication: It indicates that an operation or function resulted in an unrepresentable value.
  • NaN Comparison: NaN is not equal to any other value, including itself. This is because NaN is considered as a unique value that is not equal to anything, including itself.

    Example

                            console.log(NaN === NaN); // Output: false

Common Causes of NaN:

  • Performing arithmetic operations involving non-numeric values or undefined results in NaN.
  • Parsing non-numeric strings using parseInt() or parseFloat() may result in NaN.
  • Mathematical operations that result in undefined values, such as dividing zero by zero, result in NaN.

Checking for NaN:

You can use the isNaN() function to determine if a value is NaN.

Example

                  console.log(isNaN("Hello")); // Output: true
                  console.log(isNaN(5)); // Output: false
13. Passed by value and passed by reference

"Passed by value" and "passed by reference" are concepts that describe how data is transferred between variables or functions in programming languages, including JavaScript.

Passed by Value:
  • In "passed by value," a copy of the variable's value is passed to a function or assigned to another variable.
  • Modifications to the parameter or variable inside the function do not affect the original value outside the function.
  • This behavior is typical for primitive data types such as numbers, strings, and booleans in JavaScript.

Example

                function modifyValue(x) {
                    x = 10;
                }
                
                let y = 5;
                modifyValue(y);
                console.log(y); // Output: 5 (unchanged)
Passed by Reference:
  • In "passed by reference," the memory address (reference) of the variable is passed to a function or assigned to another variable.
  • Changes made to the parameter or variable inside the function affect the original value outside the function because they are referring to the same memory location.
  • This behavior is typical for complex data types such as arrays and objects in JavaScript.

Example

                function modifyArray(arr) {
                    arr.push(4);
                }
                
                let myArray = [1, 2, 3];
                modifyArray(myArray);
                console.log(myArray); // Output: [1, 2, 3, 4] (modified)
