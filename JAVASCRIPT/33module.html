JavaScript Modules

In JavaScript, modules are a way to encapsulate and reuse code across different parts of an application. Modules help in organizing code, improving maintainability, and avoiding global namespace pollution. JavaScript has different module systems, each serving different purposes and use cases.

  • JavaScript modules allow you to break up your code into separate files.
  • This makes it easier to maintain a code-base.
  • Modules also rely on type="module" in the <script> tag.
  • Modules are imported from external files with the import statement.

ES6 Modules:

ES6 introduced a standardized module system for JavaScript. This is now the most common way to use modules in modern JavaScript, especially in environments that support ES6, such as modern browsers and Node.js.

Syntax:

  • export: Used to export functions, objects, or primitive values from a module.
  • import:Used to import functions, objects, or primitives from a module.

Example:

Exporting 
            // module.js
            export const myFunction = () => {
              console.log("Hello from myFunction");
            };
            
            export const myVariable = 42;
            
            export default function defaultFunction() {
              console.log("This is the default export");
            }
Importing: 
            // main.js
            import { myFunction, myVariable } from './module.js';
            import defaultFunction from './module.js';
            
            myFunction(); // Output: Hello from myFunction
            console.log(myVariable); // Output: 42
            defaultFunction(); // Output: This is the default export
