1. Difference between “ == “ and “ === “ operators (with examples)

In JavaScript, both the == (loose equality) and === (strict equality) operators are used for comparison, but they behave differently in terms of how they handle types and values.

Loose Equality (==) Operator:

  • The == operator compares the values of two operands after converting them to a common type.
  • It performs type coercion, meaning it may convert the operands to the same type before comparison.
  • If the types of the operands are different, JavaScript attempts to convert them to a common type before making the comparison.
  • This can lead to unexpected results, especially when comparing different data types.
Example 
                console.log(5 == "5"); // Output: true
                console.log(1 == true); // Output: true
                console.log(0 == false); // Output: true

Strict Equality (===) Operator:

  • The === operator compares both the values and the types of two operands.
  • It does not perform type coercion, so the values must be of the same type to be considered equal.
  • If the types of the operands are different, the === operator immediately returns false without attempting to convert them.
Example 
            console.log(5 === "5"); // Output: false
            console.log(1 === true); // Output: false
            console.log(0 === false); // Output: false

JavaScript Relational Operators:

JavaScript Relational operators are used to compare its operands and determine the relationship between them. They return a Boolean value (true or false) based on the comparison result.

The in Operator

The in operator returns true if the specified property is in the specified object.

Example 
            // Creating an object
            let car = {
            make: 'Toyota',
            model: 'Camry',
            year: 2022
            };

           // Checking if a property exists using the 'in' operator
           let hasMake = 'make' in car;
           let hasColor = 'color' in car;

           // Outputting the results
           console.log('Has make property:', hasMake); // Output: true
           console.log('Has color property:', hasColor); // Output: false

           Output:
           Has make property: true
           Has color property: false

The instanceof Operator

The instanceof operator returns true if the specified object is of the specified object type.

Example 
            // Creating a constructor function
            function Car(make, model, year) {
               this.make = make;
               this.model = model;
               this.year = year;
             }

            // Creating an instance of the Car constructor
            let myCar = new Car('Toyota', 'Camry', 2022);

           // Checking if the object is an instance of a particular class using the 'instanceof' operator
           let isCarInstance = myCar instanceof Car;

           // Outputting the result
           console.log('Is myCar an instance of Car?', isCarInstance); 
           Output: true
