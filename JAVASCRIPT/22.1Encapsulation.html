Encapsulation(Data Hiding)

Definition

ðŸ’¬ "Encapsulation means bundling data and methods together inside an object and restricting direct access to some data to protect its integrity."

ðŸ‘‰ Example (Bank Account System)

"Imagine a bank account where the balance should not be modified directly. We keep the balance private and provide methods to deposit and withdraw money safely."

ðŸ”¹ Code Reference (Verbal Explanation)

"In JavaScript, we can use private fields (#balance) to ensure the balance is not directly accessible. The deposit and withdraw methods provide controlled access."

Example

            class Person {
                constructor(name, age) {
                  this.name = name; // Public property
                  let _age = age;   // Private variable (not truly private but follows convention)
              
                  this.getAge = function() {
                    return _age;
                  };
              
                  this.setAge = function(newAge) {
                    if (newAge > 0) _age = newAge;
                    else console.log("Age must be positive!");
                  };
                }
              }
              
              const person1 = new Person("John", 25);
              console.log(person1.name);  // John (public property)
              console.log(person1.getAge()); // 25 (using method)
              person1.setAge(30);
              console.log(person1.getAge()); // 30

Note: Here, _age is not directly accessible outside the class, ensuring encapsulation.
