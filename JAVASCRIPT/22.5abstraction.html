"In JavaScript, we use classes with private fields to hide internal logic. For example, in a bank account system, users only see deposit and withdraw methods but not the internal calculations."

Example: using private fields (#) in ES6:

            class BankAccount {
                #balance;  // Private field
              
                constructor(accountNumber, balance) {
                  this.accountNumber = accountNumber;
                  this.#balance = balance;
                }
              
                deposit(amount) {
                  if (amount > 0) this.#balance += amount;
                }
              
                withdraw(amount) {
                  if (amount > 0 && amount <= this.#balance) {
                    this.#balance -= amount;
                  } else {
                    console.log("Insufficient balance!");
                  }
                }
              
                getBalance() {
                  return this.#balance;
                }
              }
              
              const myAccount = new BankAccount("123456", 1000);
              console.log(myAccount.accountNumber); // Accessible
              console.log(myAccount.getBalance());  // 1000 (Public method accessing private data)
              myAccount.deposit(500);
              console.log(myAccount.getBalance());  // 1500
              myAccount.withdraw(2000);  // Insufficient balance!

Note:The #balance field is private and can't be accessed directly.
